TAG 15 – Erstes EKS Deployment erfolgreich
------------------------------------------------------------

- EKS Cluster via Terraform gestartet
- Node Group aktiviert (t3.small, desired=1)

- Kubernetes Deployment durchgeführt:
  kubectl apply -f k8s/

- Frontend Pod erfolgreich gestartet
- Backend Pod gestartet, zunächst CrashLoopBackOff

Root Cause Analyse:

- Kubernetes Liveness / Readiness Probe schlug fehl
- Ursache: Spring Security blockierte /actuator/health
- Kubernetes erhielt HTTP 401 statt HTTP 200

Fix:

- SecurityConfig angepasst:
  .requestMatchers("/actuator/**").permitAll()

- Backend Image neu gebaut

Weiterer Fehler entdeckt:

- Docker Image Architekturproblem
- Mac M-Series baut ARM Images
- EKS Nodes laufen auf AMD64

Fehler im Cluster:

no match for platform in manifest

Fix:

docker build --platform linux/amd64

- Backend Image neu gebaut
- Image mit Git Commit SHA getaggt
- Image nach ECR gepusht
- Deployment auf neues Image aktualisiert

Verifikation:

kubectl get pods

backend 1/1 Running
frontend 1/1 Running

Ergebnis:

- Erstes vollständiges Deployment auf EKS erfolgreich
- Backend + Frontend laufen stabil im Cluster
- PostgreSQL Verbindung zu RDS erfolgreich
- Kubernetes Health Checks funktionieren
- Docker ARM → AMD64 Problem sauber gelöst
